Question 1 Amazon Bedrock Fundamentals
A developer calling ConverseStream wants the model to stop generating as soon as it produces a specific delimiter used by downstream parsing, to save output tokens. Using the unified inference configuration, which setting achieves this?
  1. AAppend the delimiter to the system prompt, which causes the runtime to truncate at that string
  2. BProvide the delimiter in inferenceConfig.stopSequences so generation halts when the sequence is produced
  3. CSet inferenceConfig.temperature to 0 so the model always stops at the delimiter
  4. DSet a low topP value to force the model to emit the delimiter early
Show answer & explanation
Correct answer: B
stopSequences in the Converse inferenceConfig tells the model to stop generating when it emits one of the specified sequences, directly saving output tokens. Temperature and topP affect sampling randomness, not stopping behavior, and putting the delimiter in the system prompt does not configure a runtime stop condition.

Correct answer: B

Correct answer: A

Question 74 Security, Cost & Operations

Question 75 Security, Cost & Operations
A workload's traffic is bursty: long idle periods with occasional short, sharp spikes that briefly exceed on-demand limits. The team wants to avoid both throttling-induced failures and paying for idle reserved capacity. Which strategy balances cost and reliability best?

  4. DProvision a small committed Provisioned Throughput tier sized to the peak and let it sit idle between spikes

Correct answer: A
On-demand with backoff and a smoothing queue charges only for tokens used and handles transient spikes, which fits bursty, mostly-idle traffic without paying for reserved capacity.
