Question 1 Amazon Bedrock Fundamentals
A latency-sensitive voice assistant must begin speaking the model's answer as soon as possible to minimize perceived response time, while still using the model-agnostic message interface. Which Bedrock runtime call is the BEST fit?
  1. AConverseStream, to receive incremental output as it is generated
  2. BCreateModelInvocationJob with near-real-time polling of the S3 output
  3. CConverse, then chunk the full text client-side for playback
  4. DInvokeModel with a reduced max token setting to force a faster full response
Show answer & explanation
Correct answer: A
ConverseStream returns tokens incrementally through the unified Converse interface, letting the assistant start playback before generation completes, minimizing time-to-first-token. Converse returns only the complete response, reducing max tokens still waits for the full body, and batch inference is asynchronous and unsuitable for real-time voice.

Question 5 Amazon Bedrock Fundamentals
