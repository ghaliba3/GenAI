Question 1 Amazon Bedrock Fundamentals
A team migrates from InvokeModel to the Converse API so they can swap between Anthropic and Amazon models without rewriting request bodies. During testing, only the Anthropic model honors a document they attach as a message content block; the Amazon Titan text model errors. What is the MOST accurate explanation?
  1. AConverse normalizes the message schema, but individual model features such as document content blocks are only available where the underlying model supports them
  2. BDocument blocks require InvokeModel; Converse silently drops them for all models
  3. CThe Amazon model requires the document to be base64-encoded in the system prompt instead
  4. DThe Converse API never supports document content blocks and the Anthropic behavior is a preview bug
Show answer & explanation
Correct answer: A
Converse gives a consistent message and inferenceConfig envelope, but capability support (documents, tool use, vision) still depends on the specific model; unsupported features return errors. Document blocks are a real Converse feature, they are not exclusive to InvokeModel, and the fix is not to smuggle content into the system prompt.

Question 75 Security, Cost & Operations
A team steadily serves a predictable, high volume of on-demand requests and repeatedly hits throttling; occasional backoff is no longer enough and latency must be guaranteed. What is the appropriate operational change?
  1. ALower max output tokens to eliminate throttling permanently
  2. BPurchase Provisioned Throughput (model units) to guarantee dedicated capacity for the steady workload
  3. CRely on cross-region inference alone to guarantee latency SLAs
  4. DSwitch entirely to batch inference for the real-time traffic
Show answer & explanation
Correct answer: B
For sustained, predictable high-volume demand needing guaranteed capacity and latency, Provisioned Throughput provides dedicated model units. Batch is asynchronous and unsuitable for real-time, reducing output tokens does not fix capacity limits, and cross-region inference improves availability/throughput but does not itself guarantee a latency SLA.
